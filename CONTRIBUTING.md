# Contributing

QuantForge changes must preserve scientific and governance controls. Open an issue before changing
the workflow, schema versions, canonical serialization, evidence requirements, audit identity, or
verdict policy. New behavior requires tests for success, rejection, tampering, and boundary cases.

Use Python 3.12 or newer, install the hash-locked runtime requirements and exact development pins,
then run `scripts/quality.sh`. Never weaken a gate merely to make a build pass. Do not include
credentials, private market data, empirical profitability claims, or modifications to the protected
sibling C++ repository.

By contributing, you agree that your contribution is licensed under Apache License 2.0.
